JS模块化

AMD(RequireJS) vs CMD(Sea.js)

为什么使用 Sea.js ？

  • 简单友好的模块定义规范：Sea.js 遵循 CMD 规范，可以像 Node.js 一般书写模块代码。
  • 自然直观的代码组织方式：依赖的自动加载、配置的简洁清晰，可以让我们更多地享受编码的乐趣。

兼容性

  • Chrome 3+ ✔
  • Firefox 2+ ✔
  • Safari 3.2+ ✔
  • Opera 10+ ✔
  • IE 5.5+ ✔

Sea.js 可运行在 Mobile 端，包括 Hybrid 模式的 App 上。理论上，Sea.js 可以运行在任何浏览器引擎上。
